Tamboli Samaj — Hostinger Deployment Guide
==========================================

STEP 1 — Upload to public_html
------------------------------
Upload the FULL GitHub repo into Hostinger public_html (not inside a subfolder).

Required at public_html/:
  app/  vendor/  database/  storage/  uploads/
  bootstrap.php  index.php  .htaccess  .env
  main/  portal/  admin/

Delete old/duplicate folders on server:
  pratibha/  portal2783/  admin.2745/  public/

STEP 2 — Choose deploy mode
---------------------------

MODE A — Recommended (3 document roots)
  Hostinger → Websites → Manage → Domains → Subdomains

  tambolisamaj.online        → public_html/main
  portal.tambolisamaj.online → public_html/portal
  admin.tambolisamaj.online  → public_html/admin

  Each folder has its own index.php + .htaccess (DirectoryIndex set).

MODE B — Fallback (1 document root)
  Point ALL domains to public_html/

  Root index.php auto-detects site/portal/admin from HTTP_HOST.
  Assets load via /portal/assets/ (handled by root .htaccess).

STEP 3 — Production .env
------------------------
  SESSION_DOMAIN=.tambolisamaj.online
  SESSION_SECURE=true
  SITE_URL=https://tambolisamaj.online
  PORTAL_URL=https://portal.tambolisamaj.online
  ADMIN_URL=https://admin.tambolisamaj.online
  DB_HOST=localhost
  DB_NAME=your_database
  DB_USER=your_user
  DB_PASS=your_password

STEP 4 — Permissions
--------------------
  storage/  → 755 or 775 (writable)
  uploads/  → 755 or 775 (writable)
  portal/uploads/  → 755 or 775 (writable)

STEP 5 — Verify
---------------
  Visit https://tambolisamaj.online/diagnose.php
  Fix any red ✘ items, then DELETE diagnose.php.

STEP 6 — Linux case sensitivity
-------------------------------
  Folders MUST be capitalized exactly:
  app/Config  app/Core  app/Controllers  app/Views
  Wrong: app/config  app/core  → 500 error on Linux

Troubleshooting
---------------
  403 Forbidden → wrong document root OR missing index.php (fixed in repo)
  500 error     → check storage/logs/ or enable APP_DEBUG briefly
  No CSS        → portal/assets/ must exist; portal doc root = public_html/portal
  DB error      → check .env DB_* on Hostinger (DB_HOST is usually localhost)
