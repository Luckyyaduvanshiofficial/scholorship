Tamboli Samaj — Multi-Site Hostinger Deployment
==============================================

Upload this entire project into Hostinger public_html:

  public_html/                    ← project root on server
  ├── app/                        ← shared PHP application
  ├── vendor/
  ├── database/
  ├── storage/                    ← writable (755/775)
  ├── uploads/                    ← writable (755/775)
  ├── .env                        ← single config (see .env.example)
  ├── bootstrap.php
  ├── main/                       ← tambolisamaj.online document root
  │   ├── index.php
  │   └── .htaccess
  ├── portal/                     ← portal.tambolisamaj.online document root
  │   ├── index.php
  │   ├── .htaccess
  │   └── assets/, uploads/, favicon, manifest, sw.js
  └── admin/                      ← admin.tambolisamaj.online document root
      ├── index.php
      └── .htaccess

Hostinger subdomain document roots (REQUIRED — fixes 403 Forbidden):
  tambolisamaj.online        → public_html/main
  portal.tambolisamaj.online → public_html/portal
  admin.tambolisamaj.online  → public_html/admin

  Hostinger: Websites → Manage → Domains → Subdomains → edit each
  → set "Document root" to the folder above (NOT public_html alone).

  If the main domain still shows 403, a fallback index.php exists at
  public_html/index.php — but subdomain roots should still be set correctly.

Cleanup on server (delete old/duplicate folders after upload):
  pratibha/, portal2783/, admin.2745/, public/ (use portal/ for assets)
  Optional: .git/, docs/, *.ps1, AGENTS.md (not needed in production)

Production .env:
  SESSION_DOMAIN=.tambolisamaj.online
  SESSION_SECURE=true
  SITE_URL=https://tambolisamaj.online
  PORTAL_URL=https://portal.tambolisamaj.online
  ADMIN_URL=https://admin.tambolisamaj.online

Shared auth: one database, one SESSION_NAME, cookie domain .tambolisamaj.online
